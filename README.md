# PO Galang Dana

A web-based pre-order system for UMKM community fundraising. Customers browse the product catalog, add items to a cart, and submit orders via WhatsApp — no account registration required.

Built for Gereja Maria Bunda Karmel, Jakarta Barat.

---

## Overview

This application serves as a lightweight storefront for coordinating pre-orders between customers and the organizing committee. The order process is intentionally frictionless: customers enter their name and WhatsApp number, and the committee handles payment confirmation and delivery coordination directly through WhatsApp.

All orders are recorded to a database in real time, allowing the admin team to monitor incoming orders, confirm payments, and manage the product catalog from a dedicated dashboard.

---

## For Business Owners

You do not need a technical background to operate this system. After initial setup:

- Products are managed through the admin dashboard at `/admin`
- Toggle availability on or off for any product in one click
- New orders appear automatically on the dashboard without refreshing the page
- Payment status (unpaid, paid, cancelled) is updated with a single button
- Contact each customer directly via WhatsApp from the admin panel

To get started, ask your developer to seed the initial product catalog and create your admin account.

### Local Setup

```bash
# 1. Clone the repository
git clone https://github.com/DavidsonRafaelK/order.id.git
cd order.id

# 2. Install dependencies
pnpm install

# 3. Configure environment variables
cp .env.example .env.local
# Fill in your Supabase credentials (see Environment Variables section below)

# 4. Run database migrations
pnpm db:migrate

# 5. Start the development server
pnpm dev
```

Open [http://localhost:3000](http://localhost:3000) to view the storefront.

### Environment Variables

Copy `.env.example` to `.env.local` and fill in the values below.

| Variable | Description | Where to Find |
|----------|-------------|---------------|
| `NEXT_PUBLIC_SUPABASE_URL` | Your Supabase project URL | Dashboard > Settings > API |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Public anon key | Dashboard > Settings > API |
| `SUPABASE_SERVICE_ROLE_KEY` | Service role key — server only, never exposed to the browser | Dashboard > Settings > API |
| `DATABASE_URL` | Transaction pooler connection string (port 6543) — used at runtime | Dashboard > Settings > Database |
| `DATABASE_URL_DIRECT` | Session pooler or direct connection (port 5432) — used for migrations only | Dashboard > Settings > Database |
| `NEXT_PUBLIC_APP_URL` | Public URL of this application | Your deployment URL |

### Database Migrations

```bash
# Generate a new migration file from schema changes
pnpm db:generate

# Apply pending migrations to the database
pnpm db:migrate

# Push schema changes directly (development only, skips migration files)
pnpm db:push

# Open Drizzle Studio to inspect and edit data visually
pnpm db:studio
```

### Admin Dashboard Setup

**Creating an admin account**

Admin accounts are created directly in Supabase — there is no self-registration:

1. Go to Supabase Dashboard > Authentication > Users
2. Click Add user > Create new user
3. Enter an email and password
4. Use those credentials to log in at `/admin/login`

**Enabling Realtime**

For live order updates to appear without page refresh, enable Realtime on the `orders` table:

Option 1 — via Supabase SQL Editor:

```sql
ALTER PUBLICATION supabase_realtime ADD TABLE orders;
```

Option 2 — via Supabase Dashboard: Database > Replication > enable INSERT and UPDATE for the `orders` table.
